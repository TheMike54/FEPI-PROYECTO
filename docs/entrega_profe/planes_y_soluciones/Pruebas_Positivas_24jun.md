# Pruebas Positivas

**Fecha real del plan:** 24-jun-2026  
**Fase del proyecto:** Validacion del flujo feliz de principio a fin

## Situacion / problema

Necesitabamos demostrar que el sistema podia recorrer un contrato completo sin depender de datos incompletos. Propusimos crear un contrato de prueba con PDF firmado y llevarlo por registro de usuarios, aprobacion, alta, programa, bitacora, avance, estimacion, revision, pago, convenio, expediente y cierre.

Decidimos crear un contrato propio para la prueba porque ya habiamos detectado bloqueo en contratos de demostracion sin PDF firmado ligado.

## Acciones realizadas

Definimos pasos con cuentas, pantallas y datos exactos para seguir el flujo feliz. Incluimos registro de usuarios, aprobacion por dependencia, alta de contrato con catalogo y programa, apertura de bitacora, captura de avances, integracion de estimacion, presentacion, revision, autorizacion, promocion a cobro, pago y consulta final.

La revision de pruebas exhaustivas complemento este plan al identificar que varios contratos demo no tenian el PDF necesario, por lo que reforzamos la recomendacion de probar el flujo positivo con un contrato creado de inicio a fin y no con datos incompletos.

## Resultado

Dejamos el plan positivo como guia imprimible para demostrar el flujo correcto. Tambien dejamos claro que debemos probar el sistema con datos completos, porque un contrato sin PDF firmado puede bloquear legalmente la integracion de estimaciones.

## Trazabilidad legal

- LOPSRM art. 46: contrato y documentos.
- LOPSRM art. 54: estimaciones, revision y pago.
- RLOPSRM art. 45-A: programa de obra.
- RLOPSRM art. 123: bitacora.
- RLOPSRM art. 132: documentacion de estimaciones.
- RLOPSRM art. 143 fr. I: amortizacion proporcional.
